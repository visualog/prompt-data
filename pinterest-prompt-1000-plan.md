## Core Selection Rules

저장 대상은 아래 조건을 모두 만족해야 한다.

1. 이미지 안에 프롬프트 텍스트가 확실히 보여야 한다.
   - `/PROMPT`, `PROMPT`, `Prompt`, `Copy prompt`, `Full prompt`, `Prompt included` 같은 라벨만으로는 부족하다.
   - 실제 이미지 생성에 사용할 수 있는 문장형 프롬프트, 설정값, 장면 설명, 스타일 설명이 이미지 내부에 포함되어야 한다.

2. 이미지 품질이 높아야 한다.
   - 저해상도, 과도하게 흐릿한 이미지, 잘린 텍스트, 읽기 어려운 작은 글씨는 제외한다.
   - 광고 배너처럼 보이거나 사이트 유입용 썸네일만 있는 이미지는 제외한다.
   - 결과 이미지와 프롬프트 텍스트가 함께 정돈된 카드형 구성을 우선한다.

3. 스타일이 최대한 겹치지 않아야 한다.
   - 같은 제작자, 같은 템플릿, 같은 레이아웃, 같은 인물 포즈가 반복되는 핀은 제한한다.
   - 하나의 시리즈에서 너무 많이 저장하지 않는다.
   - 비슷한 핀이 여러 개이면 더 선명하고 프롬프트가 잘 보이는 1개만 저장한다.

4. 보드 성격과 맞지 않는 핀은 저장하지 않는다.
   - 단순 AI 결과물만 있는 이미지
   - 프롬프트 작성법 인포그래픽
   - ChatGPT 업무 프롬프트, 마케팅 프롬프트, 글쓰기 프롬프트
   - `Comment "PROMPT"`, `DM me`, `send prompt`, `link in bio`, `Telegram` 유도형 콘텐츠
   - 강의, 템플릿 판매, 전자책, 프롬프트 번들 광고

## Batch Strategy

한 번에 1,000개를 저장하지 않고 50개 단위로 수집한다.

각 배치 절차:

1. 검색 쿼리 8-12개를 준비한다.
2. 후보 핀을 100-300개 정도 모은다.
3. 아래 기준으로 1차 제외한다.
   - 프롬프트 텍스트가 이미지 안에 없는 핀
   - 댓글 유도형 핀
   - 단순 결과 이미지
   - 너무 유사한 템플릿
4. 남은 후보를 눈으로 확인해 50개 이하로 줄인다.
5. `Prompt` 보드에 저장한다.
6. 저장 후 보드 개수를 확인한다.
7. 새로 저장된 핀 중 기준에 맞지 않는 것이 있으면 즉시 제거한다.

## Acceptance Checklist Per Pin

저장 전 각 핀을 아래 체크리스트로 판단한다.

- [ ] 이미지 내부에 실제 프롬프트 문장이 보인다.
- [ ] 프롬프트 텍스트가 읽을 수 있을 정도로 선명하다.
- [ ] 결과 이미지 품질이 높다.
- [ ] 기존 보드 핀과 시각적으로 중복되지 않는다.
- [ ] 광고, 강의, 댓글 유도, 텔레그램 유도 콘텐츠가 아니다.
- [ ] 단순 팁/인포그래픽이 아니라 이미지 생성 예시 카드다.
- [ ] 저장할 보드가 정확히 `Prompt`로 선택되어 있다.

## Duplicate Control

중복 방지를 위해 아래 방식으로 관리한다.

1. 저장 전 Pinterest가 `Pin Saved`로 표시하면 건너뛴다.
2. 같은 제작자 또는 같은 템플릿은 배치당 3-5개 이내로 제한한다.
3. 같은 인물 사진, 같은 구도, 같은 컬러 팔레트가 반복되면 더 좋은 1개만 남긴다.
4. 같은 검색어에서 연속 저장하지 않고 카테고리를 섞는다.
5. 100개 단위로 보드를 훑어 기준 미달 핀을 정리한다.

  - 일반 ChatGPT 업무 프롬프트 제외
  - Telegram 유도형 핀은 이미지 품질이 좋아도 마지막 선별 단계에서 제외

## Working Rule

수량보다 품질을 우선한다. 저장 속도가 느려지더라도 이미지 내부에 프롬프트 텍스트가 확실히 있고, 결과 이미지가 참고할 만한 수준인 핀만 추가한다.
